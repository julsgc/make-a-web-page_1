LESSON 1: Web and HTML Basics

THE WEB

The web is the network of collected documents saved on servers and on individuals computers - all connected and communicating with one another, consisting of:

  • URLS: The Uniform Resource Locator is the file location or address of a file on the web (it is the www.blahblahblah.ninja for example).
  • HTTP: The Hypertext Transfer Protocol is a standard that allows users to exchange information found on web pages. It directs your computer over the internet to the servers where the document needed for your web page is stored. There is also FTP and HTTPS.
  • WEB APPLICATIONS: Any software/application that runs in a web browser, created in various languages (Java, CSS and HTML, for example)
  • A WEB PAGE is a text document written in a web application language interpreted by your web browser to display them
HTML VOCABULARY

HTML is a web application language used to make web pages and consists of:

  • TAGS: An HTML tag is always contained within angled brackets.
    • 1. CLOSING - ones that require both a < > and < / > for example
    • 2. VOID - these do not require a closing slash tag
  • ELEMENTS: Everything within a set of opening and closing tags.
    • INLINE - keeps information in the same line
    • BLOCK - separating elements, creating an invisible block of content
  • ATTRIBUTE: The property of an HTML element. They sit inside the tag within quotations marks. For example the link to www.studiograce.net looks like this <a href="www.studiograce.net">www.studiograce.net</a>
CODING WORKS LIKE THIS

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote. For example the computer might write "I'm learning to code!"" by making the text bold like using this set of tags: <b> I'm learning to code!</b>

COMPUTERS ARE STUPID

Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax"). For example, if you forget to close a "closing tag," the computer won't be able to figure out what you had intended to make bold. This "stupidity" can be very frustrating, but it also gives programmers incredible power: if you know how to talk to a computer than you can tell it to do anything you want.

WRITING CODE

We can use various text editors to write code. Sublime Text and various other programming specific text editors can very helpful in creating a page by automatically generating the closing tags and providing references to tag names.

The steps to creating a page:

  1. Take the design and look for natural Boxes
  2. Look for common styles of various elements
  3. Write HTML
  4. Apply Styles
  5. Fix things

LESSON 2: The Roles of HTML, CSS and JAVA

HTML
(Hyper Text Markup Language)

Provides the Structure to a page

  • HTML documents contain content (what you see) and mark up (how it looks).
  • DOM (Document Object Model) is the tree like structure created by the nesting of elements in HTML. In reading the code it is a series of collapsible and expandable triangles/nodes. This works much like an org chart or family tree going form parent to child.
  • Reading HTML: in reading this code we see a series of indentations forming a wave with the left side of the window. Each indention is nested within in the one above it - the more indented, the more nested an element is.
  • Boxify your page design using Div tags: Div is a block element that allows us to implement the design of a page by looking at it as a series of boxes.
  • Labeling your boxes with relevant names helps with the styling in CSS later.
CSS
(Cascading Style Sheet, with the emphasis on Cascading)

Provides the Style to a page

  • CSS is a separate file that is referenced in your HTML code
  • CSS saves a lot of work and time by allowing us to change the look of a site by only editing a much smaller CSS file.
  • CSS defines how the various elements (positioning, layout, fonts, colors and other style information) within HTML are displayed by using selectors
  • A single CSS file can define the style for multiple HTML files, which helps programmers avoid coding repetition* of which the benefits include:
    • avoiding coding mistakes
    • smaller files
    • less typing
    • easier page style editing
    • faster page loading
    • code can be more "readable" in that there is less repetitive code to skim through
  • Cascading is the inheritance of styles by elements - the ancestor descendant "inherited" relationship of their properties. Not all properties are inherited. See this web page for reference of what is and is not inherited
  • Box Model: a box that wraps around HTML elements, and it consists of: margins, borders, padding, and the actual content. See this web page for graphic representation of one.

* Avoiding repetition is a common theme in all programming languages and one of the key tenets to "good" coding. It requires abstract thinking about the functionality of the various parts of a web page (or program). The concept is referred to as DRY (Don't Repeat Yourself). See this page for more details. Also see this page about abstraction. Further, in creating this page it has allowed me to merely define the roles of the text on the page with a common element tag for items with common roles (headings, concepts, concept titles, etc). Then in a single CSS file I can define the appearance of each role/element, and as required, change with a single line of code (size, weight, font type, color etc).
